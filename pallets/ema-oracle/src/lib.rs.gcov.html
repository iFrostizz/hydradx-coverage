      18             : //! # EMA Oracle Pallet
      19             : //!
      20             : //! ## Overview
      21             : //!
      22             : //! This pallet provides exponential moving average (EMA) oracles of different periods for price,
      23             : //! volume and liquidity for a combination of source and asset pair based on data coming in from
      24             : //! different sources.
      25             : //!
      26             : //! ### Integration
      27             : //!
      28             : //! Data is ingested by plugging the provided `OnActivityHandler` into callbacks provided by other
      29             : //! pallets (e.g. xyk pallet).
      30             : //!
      31             : //! It is meant to be used by other pallets via the `AggregatedOracle` and `AggregatedPriceOracle`
      32             : //! traits.
      33             : //!
      34             : //! When integrating with this pallet take care to use the `on_trade_weight`,
      35             : //! `on_liquidity_changed_weight` and `get_entry_weight` into account when calculating the weight
      36             : //! for your extrinsics (that either feed data into or take data from this pallet).
      37             : //!
      38             : //! ### Concepts
      39             : //!
      40             : //! - *EMA*: Averaging via exponential decay with a smoothing factor; meaning each new value to
      41             : //!   integrate into the average is multiplied with a smoothing factor between 0 and 1.
      42             : //! - *Smoothing Factor*: A factor applied to each value aggregated into the averaging oracle.
      43             : //!   Implicitly determines the oracle period.
      44             : //! - *Period*: The window over which an oracle is averaged. Certain smoothing factors correspond to
      45             : //!   an oracle period. E.g. ten minutes oracle period â‰ˆ 0.0198
      46             : //! - *Source*: The source of the data. E.g. xyk pallet.
      47             : //!
      48             : //! ### Implementation
      49             : //!
      50             : //! This pallet aggregates data in the following way: `on_trade` or `on_liquidity_changed` a new
      51             : //! entry is created for the incoming data. This then updates any existing entries already present
      52             : //! in storage for this block (for this combination of source and assets) or inserts it. Note that
      53             : //! this aggregation is NOT based on EMA, yet, it just sums the volume and replaces price and
      54             : //! liquidity with the most recent value.
      55             : //!
      56             : //! At the end of the block, all the entries are merged into permanent storage via the exponential
      57             : //! moving average logic defined in the math package this pallet depens on. There is one oracle
      58             : //! entry for each combination of `(source, asset_pair, period)` in storage.
      59             : //!
      60             : //! Oracle values are accessed lazily. This means that the storage does not contain the most recent
      61             : //! value, but the value calculated the last time it was updated via trade or liquidity change. On a
      62             : //! read the values are read from storage and then fast-forwarded (assuming the volume to be zero
      63             : //! and the price and liquidity to be constant) to the last block. Note: The most recent oracle
      64             : //! values are always from the last block. This avoids e.g. sandwiching risks. If you want current
      65             : //! prices you should use a spot price or similar.
